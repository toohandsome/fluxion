# Fluxion 分期计划表

## 1. 分期总览

| 阶段 | 目标 | 核心结果 |
| --- | --- | --- |
| 一期 | 建立最小可用闭环 | 可设计、可发布、可触发、可监控 |
| 二期 | 提升平台可复用与可运维性 | 子流程、模板、脚本、集群与观测 |
| 三期 | 建立生态扩展与高阶处理能力 | 插件体系、流式处理、调试体系 |

## 2. 一期计划

### 2.1 目标

完成一个可供真实业务试点接入的最小可用版本。

### 2.2 范围

1. 流程定义与版本管理
2. 基础前端可视化设计器
3. DAG 设计与模型编译
4. 基础执行引擎
5. 基础节点库
6. HTTP 发布
7. Quartz 调度
8. 运行监控
9. 资源连接与密钥托管
10. 操作审计
11. Starter 与 Server 两种形态

### 2.3 里程碑

| 里程碑 | 结果 |
| --- | --- |
| M1 | 完成领域模型、执行模型、数据库 DDL |
| M2 | 完成引擎核心、基础节点与持久化 |
| M3 | 完成 HTTP 发布、调度触发与管理接口 |
| M4 | 完成基础前端设计器与运行记录页面 |
| M5 | 完成一期联调、试点交付与文档收口 |

### 2.4 出口标准

1. 能保存草稿并发布版本
2. 能通过前端画布完成基础流程设计
3. 能通过 HTTP 触发流程
4. 能通过 Quartz 触发流程
5. 能查看实例和节点执行记录
6. 能通过资源表管理连接配置
7. 能通过审计表追踪关键管理操作

## 3. 二期计划

### 3.1 目标

让系统具备团队内规模化使用能力。

### 3.2 范围

1. 子流程
2. 模板库
3. 脚本节点
4. Redis 节点
5. 失败重放与补偿
6. Quartz 集群
7. 指标、链路追踪、告警通知
8. mysql/oracle/sqlserver 支持

### 3.3 出口标准

1. 流程可复用
2. 任务可集群运行
3. 运行异常可告警
4. 能支持第二种数据库

## 4. 三期计划

### 4.1 目标

完善生态化扩展能力和复杂数据处理能力。

### 4.2 范围

1. PF4J 后端插件
2. 前端插件动态加载
3. 流式与游标处理
4. 死信和背压
5. 断点调试
6. 更多数据库方言支持

### 4.3 出口标准

1. 节点能力可通过插件扩展
2. 平台可处理大结果集任务
3. 可对复杂流程进行调试和回放

## 5. 建议优先级

如果资源有限，优先顺序建议为：

1. 一期全部
2. 二期中的子流程、观测、告警
3. 三期中的插件和流式处理
