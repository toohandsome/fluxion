# Fluxion 一期实施计划

## 1. 文档定位

本文档只维护一期实施计划（里程碑、范围、出口标准与优先级），不再重复定义二期、三期范围。

二期与三期规划入口见：[../roadmap.md](../roadmap.md)。

## 2. 一期目标

完成一个可供真实业务试点接入的最小可用版本，形成“设计 -> 发布 -> 触发 -> 监控 -> 治理”闭环。

## 3. 一期范围

1. 流程定义与版本管理
2. 基础前端可视化设计器
3. DAG 设计与模型编译
4. 基础执行引擎
5. 基础节点库
6. HTTP 发布
7. Quartz 调度
8. 运行监控
9. 资源连接与密钥托管
10. 操作审计
11. Starter 与 Server 两种形态

需求基线见：[requirements.md](./requirements.md)。

## 4. 一期里程碑

| 里程碑 | 结果 |
| --- | --- |
| M1 | 完成领域模型、执行模型、数据库 DDL |
| M2 | 完成引擎核心、基础节点与持久化 |
| M3 | 完成 HTTP 发布、调度触发与管理接口 |
| M4 | 完成基础前端设计器与运行记录页面 |
| M5 | 完成一期联调、试点交付与文档收口 |

## 5. 一期出口标准

1. 能保存草稿并发布版本
2. 能通过前端画布完成基础流程设计
3. 能通过 HTTP 触发流程
4. 能通过 Quartz 触发流程
5. 能查看实例和节点执行记录
6. 能通过资源管理维护连接配置
7. 能通过审计记录追踪关键管理操作

## 6. 建议实施顺序

1. `model + spi + engine`
2. `persistence + schema`
3. `runtime-api + admin-api`
4. `starter + server`
5. `frontend + 联调 + 验收`
