# Fluxion 路线图

## 总览

Fluxion 按三个阶段推进。

## 一期

目标：建立最小可用闭环。

- 流程定义与版本管理
- 基础前端可视化设计器
- HTTP 发布与运行时触发
- Quartz 调度
- 核心执行引擎与基础节点
- 监控、审计

详细文档：

- [一期需求文档](../phase-1/requirements.md)
- [一期技术方案](../phase-1/technical-solution.md)
- [一期计划](../phase-1/plan.md)

## 二期

目标：提升复用性与运维能力。

- 子流程
- 模板库
- 脚本节点
- Redis 节点
- 集群调度
- 指标、链路追踪与告警

当前状态：

- 二期详细文档尚未正式展开。

参见：[二期占位说明](../phase-2/README.md)

## 三期

目标：建立生态扩展与高阶处理能力。

- 插件体系
- 前端插件动态加载
- 流式处理
- 断点调试
- 更多数据库方言支持

当前状态：

- 三期详细文档尚未正式展开。

参见：[三期占位说明](../phase-3/README.md)
